Introduction

This style guide defines the visual standards and design patterns for all Smart Health Hub documentation pages. Following these guidelines ensures a consistent, professional appearance across the platform's documentation, making it easier for users to navigate and understand the content.

All documentation pages are built using HTML with Tailwind CSS for styling. Code snippets use Highlight.js for syntax highlighting.

Layout & Structure

All documentation pages follow a consistent layout pattern for ease of navigation and readability.

Page Structure

Header: Page title, back links, and navigation
Introduction: Brief overview of the service or topic
Sidebar: Table of contents
Main content: Detailed information divided into sections

Grid System

The documentation uses Tailwind's grid system for responsive layouts:

  • Main layout: grid grid-cols-1 lg:grid-cols-4 gap-8
  • Sidebar: lg:col-span-1
  • Main content: lg:col-span-3
  • Card grids: grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6

Spacing

  • Section spacing: mb-8 (margin-bottom: 2rem)
  • Title spacing: mb-4 (margin-bottom: 1rem)
  • Paragraph spacing: mb-4 (margin-bottom: 1rem)
  • List item spacing: space-y-2 (gap: 0.5rem)
  • Card padding: p-6 (padding: 1.5rem)

Code Formatting

Code examples are crucial in technical documentation. Follow these guidelines for consistent code formatting:

Syntax Highlighting

Use Highlight.js for syntax highlighting. Specify the language in the class of the code element: 

<pre><code class="language-javascript">
// Your JavaScript code here
</code></pre>

Supported Languages

Common languages used in documentation:

  • language-javascript - For JavaScript/TypeScript code
  • language-java - For Java code
  • language-python - For Python code
  • language-html - For HTML markup
  • language-css - For CSS styles
  • language-json - For JSON data
  • language-sql - For SQL queries
  • language-bash - For bash/shell commands
  • language-http - For HTTP requests/responses

Best Practices

  • Keep code examples concise and focused on the specific concept
  • Include comments to explain complex or non-obvious parts
  • Use consistent spacing and indentation (2 spaces for most languages)
  • For command-line instructions, show the output when relevant
  • For HTTP requests, include both the request and response

Responsive Design

All documentation pages should be responsive, working well on mobile, tablet, and desktop devices:

Responsive Guidelines

  • Use Tailwind's responsive prefixes (sm:, md:, lg:) for different screen sizes
  • On mobile, use single-column layouts
  • On desktop, use multi-column layouts with sidebar navigation
  • Ensure tables are wrapped in overflow-x-auto for horizontal scrolling on small screens
  • Use appropriate font sizes that are readable on all devices
  • Ensure touch targets are large enough (minimum 44px) for mobile users

Responsive Component Example

<!-- This will be a single column on mobile and 2 columns on md screens and above -->
<div class="grid grid-cols-1 md:grid-cols-2 gap-6">
  <div class="bg-white shadow-md rounded-lg p-6">
    <h3 class="text-lg font-medium">Column 1</h3>
    <p class="text-gray-700">Content for column 1...</p>
  </div>

    <h3 class="text-lg font-medium">Column 2</h3>
    <p class="text-gray-700">Content for column 2...</p>

</div>

Accessibility

Ensure documentation is accessible to all users, including those with disabilities:

Accessibility Guidelines

  • Use semantic HTML elements (h1, h2, ul, etc.) appropriately
  • Maintain sufficient color contrast between text and background (WCAG AA compliance)
  • Include alternate text for all images and diagrams
  • Ensure keyboard navigability for all interactive elements
  • Use ARIA attributes where appropriate for complex interactive components
  • Provide descriptive text for links (avoid "click here")
  • Use tables appropriately with headers and proper structure

Accessible Link Example

<!-- Good: Descriptive link text -->
<a href="/docs/fhir-service" class="text-blue-600 hover:underline">
  View FHIR Service Documentation
</a>

<!-- Bad: Non-descriptive link text -->
<a href="/docs/fhir-service" class="text-blue-600 hover:underline">
  Click here
</a>
